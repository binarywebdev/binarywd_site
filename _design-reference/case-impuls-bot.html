[ 01 ] / context

Бот без интерфейса — это не продукт, а скрипт.

До v2 у нас был рабочий движок на Python: подключение к Binance, исполнение, лог в текстовый файл. Стратегия работала, но запускать её можно было только из терминала, читать результаты — глазами по CSV.

Нужно было превратить движок в продукт: дать визуальный конструктор робота, бэктест в браузере, мониторинг сделок с алертами в Telegram, и админ-панель для управления пользователями и подписками.

[ 02 ] / problem ↔ solution

Три узких места.

Самые болезненные точки — там же, где живёт деньги. Каждой назначили ответственное решение и метрику успеха.

01 · problem

Параметры робота хранились как TEXT — форма показывала дефолты, а бот торговал реальными значениями.

02 · problem

Бэктест занимал часы и шёл в одном потоке — невозможно итерировать стратегию.

03 · problem

Stale data на mark_price ломала naked-check и приводила к ложным алертам.

01 · solution

JSONB-схема с миграцией + строгая типизация params на бэке и фронте.

02 · solution

Распараллелили бэктест по символам, добавили progress-WS — 8× быстрее на той же инфре.

03 · solution

Hard timeout 5s на premiumIndex + 120s grace при старте — конец false STALE.
